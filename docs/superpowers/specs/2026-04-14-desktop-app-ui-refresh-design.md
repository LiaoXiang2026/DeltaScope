# DeltaScope 桌面端界面重设计说明
**日期**: 2026-04-14  
**范围**: `frontend/src/App.tsx` 与各页面视觉重构，不改变核心业务功能

---

## 1. 目标

将当前偏普通后台表单风格的桌面端，重构为一套更贴合 Analyze / Review 工作流的“冷静型指挥台”界面。

本次重设计不改变功能边界，只优化：

- 信息架构
- 布局层级
- 视觉风格
- 状态反馈
- 报告与结果展示体验

---

## 2. 目标用户与使用场景

主要用户：

- 开发
- 测试
- 不熟命令行的内部协作成员

典型场景：

- 发起 Git 缺陷分析
- 发起代码变更评审
- 阅读风险报告
- 配置和维护模型连接信息

界面需要支持“高频、快速、连续”的内部使用，而不是一次性展示型页面。

---

## 3. 设计方向

### 方向名称
冷静型指挥台

### 气质
- 专业
- 冷静
- 可信
- 带轻度风险审查感

### 颜色策略
- 主底色：石墨灰
- 强调色：暗红、铜色
- 中性色需带轻微暖色偏移，避免偏蓝

### 明确排除
- 深灰蓝底色
- 钴蓝 / 矿物青强调色
- 霓虹赛博感
- 紫蓝渐变
- 大量玻璃拟态
- 常规 SaaS 卡片栅格

---

## 4. 布局方案

采用“双栏指挥台”结构。

### 左侧：任务控制台
负责放置：

- 页面标题
- 当前页面说明
- 参数输入区
- 开关项
- 运行按钮
- 当前状态与反馈

设计目标：

- 更像任务发起器，而不是普通表单
- 层级清晰，强调执行动作
- 在 Analyze / Review 中都能快速进入状态

### 右侧：结果舞台
负责放置：

- Analyze 的 dashboard 结果
- Review 的 Markdown 报告
- 对应的空状态与完成状态

设计目标：

- 让结果成为视觉中心
- Analyze 更像监控面板
- Review 更像内部审查报告

### 顶部：窄头部
只承担：

- 产品标识
- 当前页面定位
- 当前运行状态

避免顶部承担过多导航和说明内容。

---

## 5. 页面级设计

### 5.1 Config 页面

改造成“连接配置台”。

表现方式：

- 不再是松散表单
- 增强字段层级与说明关系
- 保存状态有更明确的反馈
- 让配置页看起来像“系统连接设置”，而不是普通资料填写

### 5.2 Analyze 页面

改造成“分析任务台 + 结果剧场”。

表现方式：

- 左侧输入区强调任务参数组织
- 右侧 dashboard 占据主视觉
- 空状态直接告诉用户结果将展示在哪里
- 运行中状态突出“分析正在进行”

### 5.3 Review 页面

改造成“评审发起器 + 审查报告阅读器”。

表现方式：

- 左侧负责 repo/base/head 和模型配置补充
- 右侧 Markdown 按报告阅读体验重新排版
- 强化结论、列表、代码块和分段层级
- 让用户更像在读正式评审输出，而不是网页文本

---

## 6. 视觉细节

### Typography
- 采用更有个性的标题字重和更紧凑的层级
- 保持正文可读性，不使用花哨字体
- 避免“开发者工具 = 全局等宽字体”的偷懒做法

### 空间与分区
- 使用明显的分区边界和节奏变化
- 控制台和结果区之间要有明确主次
- 不使用重复卡片堆砌来制造层级

### 状态语义
强化以下状态的视觉表达：

- Ready
- Running
- Complete
- Failed

状态表达以标签、边框、背景层次和强调色完成，不依赖炫光。

### Motion
- 只保留必要的进入和状态切换动效
- 以 opacity / transform 为主
- 不做夸张弹跳或装饰性动画

---

## 7. 技术边界

本次实现仅限前端层：

- `frontend/src/App.tsx`
- `frontend/src/pages/ConfigPage.tsx`
- `frontend/src/pages/AnalyzePage.tsx`
- `frontend/src/pages/ReviewPage.tsx`
- `frontend/src/styles.css`

允许增加少量纯前端辅助组件或样式抽取，但不重写业务 API。

不在本次范围内：

- 改动 Go 端业务逻辑
- 新增后端接口
- 调整 Analyze / Review 功能规则

---

## 8. 成功标准

1. 界面明显摆脱“普通后台表单页”观感。
2. Analyze 和 Review 的结果区成为视觉中心。
3. 配色与结构能传达风险审查工具的专业感。
4. 操作仍然清晰，不因设计增强而降低可用性。
5. 在桌面端窗口中保持良好的可读性和布局稳定性。
